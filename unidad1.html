UNIDAD 1: Introducción a las estructuras de datos

1.1 Clasificación de las estructuras de datos

Las estructuras de datos se clasifican según su propósito, diseño y forma de acceso:

  • Primitivas: Estas son proporcionadas por los lenguajes de programación e incluyen datos básicos como enteros, flotantes, caracteres y booleanos. Son los bloques fundamentales de cualquier programa.
  • No primitivas: Estas son estructuras más complejas que se construyen a partir de las primitivas. Incluyen:
    • Lineales: Se organizan de manera secuencial, como arreglos, listas, pilas y colas.
    • No lineales: Los datos están organizados jerárquicamente o conectados por relaciones complejas, como árboles y grafos.
  • Estáticas: Su tamaño se define antes de la ejecución del programa y no puede cambiar, como los arreglos tradicionales.
  • Dinámicas: Pueden crecer o reducirse durante la ejecución, ajustándose a las necesidades del programa, como las listas enlazadas.

1.2 Tipos de datos abstractos (TDA)

Un TDA es una representación matemática o lógica de una colección de datos, junto con las operaciones definidas para ellos. Los TDA definen qué puede hacerse con los datos sin especificar cómo se implementan estas operaciones.

  • Listas: Permiten almacenar datos de forma ordenada y acceder a ellos por su índice.
  • Pilas: Permiten apilar elementos y realizar operaciones únicamente en la parte superior de la estructura.
  • Colas: Gestionan elementos en un orden de llegada y salida específico.

1.3 Ejemplos de TDA's

Los TDA se implementan para resolver problemas específicos, como la gestión de procesos en sistemas operativos mediante colas, o el manejo de expresiones aritméticas utilizando pilas.

1.4 Manejo de memoria

El manejo eficiente de la memoria es crucial en cualquier programa:

  • Memoria estática: Se asigna de manera fija al inicio del programa y es utilizada por variables globales y locales declaradas explícitamente. Es más rápida pero menos flexible.
  • Memoria dinámica: Se asigna y libera durante la ejecución del programa, permitiendo un uso más eficiente. En Java, el recolector de basura automatiza la gestión de memoria, liberando espacio utilizado por objetos que ya no tienen referencias activas.

1.5 Análisis de algoritmos

El análisis de algoritmos permite evaluar la eficiencia de un programa:

  • Complejidad temporal: Describe cómo el tiempo de ejecución del algoritmo cambia en función del tamaño de la entrada. Se mide en términos de notación Big-O.
  • Complejidad espacial: Calcula la cantidad de memoria requerida para ejecutar el algoritmo, incluyendo el espacio para variables, estructuras temporales y recursividad.
  • Eficiencia: Considera tanto el tiempo como la memoria utilizados para evaluar qué tan óptimo es un algoritmo en comparación con otros para resolver el mismo problema.
